## What I work on

- regulated data systems across AWS and Azure
- metadata-driven ETL and lakehouse patterns

- analytics delivery for operators, analysts, and clinical stakeholders

## Engineering principles

- make the safe path the default path: governance, quality, and lineage belong in the design
- favor repeatable framework patterns over one-off pipeline heroics
- optimize for business-ready outputs, not just technically correct transformations
- treat testing, observability, and cost control as core engineering work
